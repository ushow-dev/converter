# ADR-005: Мульти-разрешение HLS (360p / 480p / 720p)

**Статус:** accepted
**Дата:** 2024-01-01
**Автор:** AI (зафиксировано при аудите 2026-03-16)

---

## Контекст

Видеоконтент доставляется через HLS (HTTP Live Streaming).
Нужно определить набор разрешений для адаптивного стриминга (ABR).

## Рассмотренные варианты

### Вариант A: Одно разрешение (720p)
- Плюсы: меньше места, быстрее конвертация
- Минусы: нет адаптивности — плохо на медленном соединении

### Вариант B: 360p / 480p / 720p (текущий выбор)
- Плюсы: охватывает большинство сценариев, ABR работает, разумный объём
- Минусы: нет 1080p / 4K, три прогона FFmpeg

### Вариант C: 360p / 480p / 720p / 1080p
- Плюсы: полное качество для HD контента
- Минусы: значительно дольше конвертация, больше места

## Решение

Выбран **Вариант B: три уровня**.

| Разрешение | Видео | Аудио |
|---|---|---|
| 360p | 800 kbps | 128 kbps |
| 480p | 1400 kbps | 128 kbps |
| 720p | 2800 kbps | 192 kbps |

Сегменты по 10 секунд. Кодек H.264 (libx264) + AAC — максимальная совместимость с hls.js.

## Последствия

### Положительные
- hls.js автоматически выбирает качество по скорости сети
- Разумное время конвертации при `CONVERT_CONCURRENCY=1`
- Хорошая совместимость (Safari, Chrome, Firefox)

### Отрицательные / компромиссы
- 4K и 1080p контент ограничен 720p
- Три прогона FFmpeg (можно оптимизировать через `-filter_complex`)

### Риски
- При большом количестве 4K источников — потеря качества заметна

## Ревизия

Пересмотреть при:
- Запросах на 1080p от пользователей
- Переходе на аппаратное кодирование (nvidia, vaapi) — тогда 1080p бесплатен по времени
- Изменении целевой аудитории (мобильные устройства → достаточно 480p max)
