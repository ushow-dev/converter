# Admin Panel — Документация

## Обзор

Admin Panel — веб-интерфейс для управления медиа-библиотекой. Написан на Next.js 14 с TypeScript и Tailwind CSS. Доступен на порту `3000` (проксируется через nginx/docker). Все запросы к API проксируются через Next.js rewrites → Go API (без CORS).

**Аутентификация:** JWT, хранится в localStorage. При отсутствии токена — редирект на `/login`.

---

## Страницы

### `/login` — Авторизация

Вход по email/паролю (`ADMIN_EMAIL` / `ADMIN_PASSWORD` из `.env`). При успехе — JWT сохраняется в localStorage, редирект на главную.

---

### `/` — Каталог фильмов

Список всех добавленных фильмов из таблицы `movies`. Данные загружаются через `GET /api/admin/movies`.

Отображает:
- Постер / заглушка
- Название фильма
- Год
- Кнопка «Смотреть» (ссылка на плеер)

---

### `/search` — Поиск релизов

Поиск по индексерам (Prowlarr). Вводится название фильма, результаты приходят из `GET /api/admin/search`.

Каждый результат:
- Название релиза
- Размер файла
- Сидеры / личи
- Индексер
- Кнопка «Скачать» → создаёт задачу `POST /api/admin/jobs` с `source_type=torrent`

---

### `/upload` — Загрузка фильма

Две вкладки: **Локальная загрузка** и **Удалённый каталог**.

#### Вкладка «Локальная загрузка»

Загрузка файла со своего компьютера напрямую в систему.

Поля формы:
- **Название** (обязательное) — название фильма для базы данных
- **IMDb ID** (опциональное) — можно найти через кнопку поиска TMDB
- **TMDB ID** (опциональное) — можно подтянуть автоматически по названию фильма в TMDB
- **Файл** (обязательное) — видеофайл (.mkv, .mp4 и др.)

Кнопки поиска:
- При вводе TMDB ID → `GET /api/admin/movies/tmdb/{id}` → автозаполнение названия и IMDb ID
- Поиск по названию → `GET /api/admin/movies/tmdb/search?q=...` → список результатов

Флоу загрузки:
1. Файл отправляется `POST /api/admin/jobs/upload` (multipart/form-data)
2. API сохраняет файл в `/media/downloads/{job_id}/`
3. Задача попадает прямо в `convert_queue` (минуя downloader)
4. Ссылка на задачу появляется после успешного ответа

**Ограничение:** до 50 ГБ на файл.

---

#### Вкладка «Удалённый каталог»

Просмотр и загрузка фильмов с HTTP-сервера с Apache/Nginx autoindex.

**Шаг 1 — Обзор каталога**

Вводится URL директории (например `http://example.com/movies/`), нажимается «Обзор».

Запрос: `GET /api/admin/remote-browse?url=<encoded_url>`

API обходит:
- Корневую директорию → находит поддиректории
- Каждую поддиректорию → ищет видеофайлы (`.mkv .mp4 .avi .mov .m4v .ts .m2ts`) и субтитры (`.srt`)
- Возвращает список фильмов с названием, размером, ссылкой на видеофайл и списком субтитров

**Шаг 2 — Выбор и загрузка**

Таблица результатов:
| Чекбокс | Название | Размер | Субтитры |  Статус |
|---------|----------|--------|----------|---------|
| ✓ | Inception.mkv | 2.1 G | SRT | — |

- Фильмы без видеофайла — серые, недоступны для выбора
- «Выбрать все» — заголовочный чекбокс
- При нажатии «Скачать выбранные» — все выбранные фильмы скачиваются **параллельно**

**Флоу загрузки каждого фильма:**

1. Фронтенд вызывает `POST /api/admin/jobs/remote-download` с `{url, filename}`
2. API:
   - Извлекает название и год из имени файла (regex `^(.+?)\s*\((\d{4})\)`)
   - Ищет фильм на TMDB по названию+году (best-effort, не блокирует при ошибке)
   - Создаёт задачу в БД (`source_type=http`)
   - Публикует в `remote_download_queue`
3. Воркер:
   - Скачивает файл по HTTP в `/media/downloads/{job_id}/`
   - Публикует задачу в `convert_queue`
4. Конвертер обрабатывает файл стандартным путём

Статус каждого фильма в таблице обновляется в реальном времени:
- *(пусто)* — не начато
- `Добавление…` — запрос отправлен
- `✓ В очереди →` (ссылка на `/jobs/{id}`) — успешно поставлено в очередь
- `✗ Ошибка` — сбой при создании задачи

После появления хотя бы одной задачи в очереди — кнопка «Открыть задания» для перехода на `/jobs`.

---

### `/jobs` — Список задач

Таблица всех задач с фильтрацией по статусу. Данные: `GET /api/admin/jobs`.

Столбцы:
- ID задачи
- Название / источник
- Статус (`queued` / `in_progress` / `completed` / `failed`)
- Прогресс (%)
- Дата создания
- Действия (удалить)

Статусы с цветовой индикацией:
- `queued` — серый
- `in_progress` — синий
- `completed` — зелёный
- `failed` — красный

---

### `/jobs/[jobId]` — Детали задачи

Детальный просмотр одной задачи. Данные: `GET /api/admin/jobs/{jobId}`.

Отображает:
- Текущий статус и прогресс
- Источник (тип + ссылка/путь)
- Временны́е метки
- Связанный фильм (если задача завершена)
- Ошибки (если `failed`)
- Ссылка на плеер (если есть asset)
- Миниатюра постера (если есть)

---

### `/queue` — Состояние очередей

Мониторинг длины очередей Redis:
- `download_queue` (торренты)
- `remote_download_queue` (HTTP-загрузки)
- `convert_queue` (конвертация)

## API endpoints, используемые фронтендом

| Метод | URL | Назначение |
|-------|-----|-----------|
| `POST` | `/api/admin/auth/login` | Авторизация |
| `GET` | `/api/admin/movies` | Список фильмов |
| `GET` | `/api/admin/movies/{id}/thumbnail` | Постер фильма |
| `GET` | `/api/admin/movies/tmdb/{tmdbId}` | Поиск метаданных TMDB по ID |
| `GET` | `/api/admin/movies/tmdb/search?q=...` | Поиск фильма на TMDB по названию |
| `PATCH` | `/api/admin/movies/{id}` | Обновить IMDb/TMDB ID, название |
| `GET` | `/api/admin/search?query=...` | Поиск релизов через Prowlarr |
| `POST` | `/api/admin/jobs` | Создать задачу (torrent) |
| `POST` | `/api/admin/jobs/upload` | Загрузить файл локально |
| `POST` | `/api/admin/jobs/remote-download` | Поставить HTTP-загрузку в очередь |
| `GET` | `/api/admin/jobs` | Список задач |
| `GET` | `/api/admin/jobs/{jobId}` | Детали задачи |
| `GET` | `/api/admin/jobs/{jobId}/thumbnail` | Постер задачи |
| `DELETE` | `/api/admin/jobs/{jobId}` | Удалить задачу |
| `GET` | `/api/admin/remote-browse?url=...` | Обзор HTTP-каталога |

## Конфигурация

| Переменная | Назначение |
|-----------|-----------|
| `ADMIN_EMAIL` | Email администратора |
| `ADMIN_PASSWORD` | Пароль (или `ADMIN_PASSWORD_HASH` bcrypt) |
| `JWT_SECRET` | Секрет для подписи JWT |
| `TMDB_API_KEY` | Ключ TMDB API (для поиска метаданных) |
| `API_URL` | Внутренний адрес Go API (docker: `http://api:8000`) |

JWT хранится в localStorage браузера. Срок действия — 1 час.
