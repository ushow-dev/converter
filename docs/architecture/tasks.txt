Задача: реализовать Phase 1 (Compose and Infrastructure Baseline) проекта Media Processing Admin System.

## Цель
Подготовить и внедрить инфраструктурный baseline на Docker Compose для single-host deployment в соответствии с существующей архитектурой и roadmap.

## Обязательные входные документы (изучи в этом порядке)
1) docs/roadmap/implementation-plan.md
   - Source of truth: раздел Phase 1, задачи P1-01...P1-09, зависимости и DoD.
2) docs/architecture/modules/deployment-compose.md
   - Целевая модель compose: сервисы, сети, volumes, healthchecks, restart policies.
3) docs/architecture/00-system-overview.md
   - Общая архитектура и связи компонентов (включая Prowlarr).
4) docs/contracts/api-admin-player.md
   - Требования к доступности API и env/публичным интерфейсам.
5) docs/contracts/job-payloads.md
   - Требования к очередям и runtime-зависимостям.
6) docs/architecture/modules/core-api.md
   - Инфраструктурные зависимости API (Postgres/Redis/Prowlarr).
7) docs/architecture/modules/worker.md
   - Инфраструктурные зависимости Worker (Redis/qBittorrent/media volume/ffmpeg).

## Scope (строго)
Реализовать только Phase 1:
- docker-compose baseline
- .env.example
- сеть/volumes
- healthchecks
- depends_on с readiness
- ограничение внешнего доступа
- restart policy и лог-ротация
- smoke-check документ

Не реализовывать бизнес-логику API/Worker/UI (это Phase 2+).

## Нефункциональные ограничения
- Prowlarr обязателен как сервис в compose.
- Внешние порты публиковать только для frontend (и опционально nginx).
- postgres/redis/qbittorrent/prowlarr должны быть доступны только во внутренней сети.
- Конфигурация должна быть переносимой на сервер (без локальных хардкодов путей/секретов).

## Ожидаемые артефакты
- docker-compose.yml
- .env.example
- docs/roadmap/phase1-smoke-check.md
- (при необходимости) точечные обновления docs/architecture/modules/deployment-compose.md, если реализация выявит расхождения
