# Модуль: Local Storage

## Назначение

Предоставляет локальное файловое хранилище для:

- сырого скачанного контента;
- временных артефактов конвертации;
- готовых файлов для выдачи плееру.

## Границы ответственности

Входит в модуль:

- структура директорий на volume;
- lifecycle файлов (создание, перемещение, очистка temp);
- политики доступа контейнеров к каталогам.

Не входит в модуль:

- принятие бизнес-решений о запуске задач;
- долговременная метаинформация (она в БД);
- CDN/object storage функциональность.

## Интерфейсы и зависимости

Входящие:

- Worker пишет и читает файлы;
- API читает метаданные готовых артефактов;
- qBittorrent пишет скачанные данные.

Исходящие:

- при необходимости Nginx читает готовые файлы для stream/download.

## Контракты данных

Базовая структура:

- `/media/downloads`;
- `/media/converted`;
- `/media/temp`.

Файловый мета-контракт:

- `job_id` включается в путь или имя;
- атомарное перемещение `temp -> converted`;
- совместимость с проверкой контрольных сумм.

## Отказоустойчивость

- Атомарные операции перемещения финального файла.
- Периодическая cleanup-задача для `temp`.
- Проверка доступности volume в health check worker.
- Резерв по диску (минимальный free space threshold).

## Наблюдаемость и безопасность

- Метрики: занятое место, скорость роста, число cleanup ошибок.
- Логирование операций с файлами (create/move/delete) с `job_id`.
- Ограничение прав записи для сервисов, которым нужен только read.
- Запрет доступа к media volume извне docker-host без явной политики.

## Точки расширения под сериалы

- Path builder не хардкодит тип контента; строит путь через `content_type`.
- Закладывается поддержка вложенных путей для будущих эпизодов (`series/<id>/season/<n>/...`) без активации на текущем этапе.
