# Модуль: Torrent Client (qBittorrent)

## Назначение

Исполняет фактическое скачивание контента по torrent/magnet и предоставляет API для мониторинга прогресса.

## Границы ответственности

Входит в модуль:

- добавление torrent задач;
- контроль состояния загрузки;
- запись данных в локальный volume.

Не входит в модуль:

- бизнес-логика оркестрации pipeline;
- принятие решений по retry/приоритетам на уровне продукта;
- конвертация медиа.

## Интерфейсы и зависимости

Входящие:

- HTTP/Web API вызовы от Worker.

Исходящие:

- запись данных в `media/downloads`.

Зависимости:

- credentials для API доступа;
- docker network и shared volume.

## Контракты данных

Минимальный контракт интеграции:

- `add_torrent(source_ref, save_path, category, tags)`;
- `get_status(torrent_id)` с прогрессом, скоростью, ошибкой;
- `pause/resume/remove` при операционных сценариях.

Привязка к domain job:

- mapping `job_id <-> torrent_id` хранится в Core API/БД.

## Отказоустойчивость

- Повторная инициализация torrent-сессии после рестарта.
- Таймауты на API qBittorrent и ограничение числа повторов.
- Reconcile loop в worker: восстановление статуса по `torrent_id`.

## Наблюдаемость и безопасность

- Логирование запросов к qBittorrent на уровне worker.
- Health check контейнера и readiness API endpoint.
- Веб-интерфейс qBittorrent не публикуется наружу.
- Секреты хранятся в env/secrets, не в коде.

## Точки расширения под сериалы

- Использование абстракции `DownloadProvider` позволяет заменить backend без изменений API/доменного слоя.
- Для сериалов будет расширен только планировщик задач, а не контракт взаимодействия с qBittorrent.
