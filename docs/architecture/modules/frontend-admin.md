# Модуль: Frontend Admin (Next.js)

## Назначение

Предоставляет административный UI для:

- поиска контента;
- запуска задач скачивания/конвертации;
- просмотра статусов, истории и логов.

## Границы ответственности

Входит в модуль:

- UI и UX для операторов;
- аутентификация администратора;
- вызовы API и отображение состояний.

Не входит в модуль:

- бизнес-правила обработки медиа;
- выполнение скачивания/конвертации;
- прямой доступ к Redis, qBittorrent и файловой системе.

## Интерфейсы и зависимости

Входящие:

- HTTPS-запросы администратора в браузере.

Исходящие:

- HTTP API к `Go Core API`.

Зависимости:

- JWT token flow;
- конфигурация API endpoint через env.

## Контракты данных

Основные DTO со стороны UI:

- `SearchRequest` (`query`, `content_type=movie`, `filters`);
- `SearchResultItem` (`external_id`, `title`, `quality`, `seeders`, `size`);
- `CreateJobRequest` (`content_type`, `source_type`, `source_ref`, `priority`);
- `JobView` (`job_id`, `status`, `progress_percent`, `error_code`, `updated_at`).

## Отказоустойчивость

- Повторные запросы для idempotent GET.
- Для POST на создание задач — client-generated `request_id` для защиты от дублей.
- Обработка деградации API: fallback состояния "временно недоступно", без потери контекста UI.

## Наблюдаемость и безопасность

- Логи frontend ошибок и correlation id запроса.
- JWT хранится безопасно (httpOnly cookie или эквивалентная стратегия).
- Роуты admin UI защищены middleware авторизации.
- Ограничение доступа по роли `admin`.

## Точки расширения под сериалы

- UI-компоненты используют `content_type` как параметр фильтра, не как хардкод.
- Таблицы результатов и задач поддерживают расширяемую колонку `sub_items` (позже для эпизодов).
- Форма запуска задачи допускает будущие режимы (`single`, `batch`) без изменения базового layout.
