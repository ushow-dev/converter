# Шпаргалка по проекту `converter`

## 1) Быстрый локальный старт (API + админка + worker)

### Подготовка `.env`

```bash
cd /Users/robospot/prj/cleaner/converter
cp .env.example .env
```

Минимум, что нужно поменять в `.env`:

```env
POSTGRES_PASSWORD=yourpassword
JWT_SECRET=<вывод openssl rand -hex 32>
PLAYER_API_KEY=any-random-string
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=yourpassword
QBITTORRENT_PASSWORD=yourpassword
```

Сгенерировать `JWT_SECRET`:

```bash
openssl rand -hex 32
```

`PROWLARR_API_KEY` можно оставить временным до первого входа в UI Prowlarr.

### Сборка и запуск

```bash
# Сборка всех основных сервисов
docker compose build api worker frontend

# Запуск всего стека
docker compose up -d

# Проверка статуса
docker compose ps
```

Ожидаемо через ~60 секунд: `postgres`, `redis`, `prowlarr`, `qbittorrent`, `api`, `worker`, `frontend` в статусе `healthy`.

## 2) Первичная настройка сервисов

### Prowlarr

Открыть UI:

- уже опубликован: `http://localhost:9696`
- если порт не опубликован в вашем compose, временно:

```bash
docker compose run --rm --service-ports -p 9696:9696 prowlarr
```

Дальше:

1. Пройти wizard.
2. Добавить индексеры.
3. Взять `API Key` в `Settings -> General`.
4. Вставить в `.env`:

```env
PROWLARR_API_KEY=<ключ>
```

5. Перезапустить API:

```bash
docker compose restart api
```

### qBittorrent

Открыть UI:

- обычно `http://localhost:8080`
- если порт не опубликован, временно:

```bash
docker compose run --rm --service-ports -p 8080:8080 qbittorrent
```

После входа поменять пароль на значение из `.env` (`QBITTORRENT_PASSWORD`).

## 3) Проверка работоспособности

### Admin UI

- `http://localhost:3000`
- вход: `ADMIN_EMAIL` / `ADMIN_PASSWORD`

### Health checks

```bash
# API readiness (через frontend rewrite)
curl http://localhost:3000/api/health/ready

# API liveness
curl http://localhost:3000/api/health/live

# Worker health
docker compose exec worker curl -s http://localhost:8001/health
```

### Логин в API и получение токена

```bash
curl -s -X POST http://localhost:3000/api/admin/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","password":"yourpassword"}' | jq .
```

### Поиск (нужен рабочий Prowlarr)

```bash
TOKEN="eyJ..."  # токен из логина

curl -s "http://localhost:3000/api/admin/search?query=inception&content_type=movie&limit=5" \
  -H "Authorization: Bearer $TOKEN" | jq .items[0]
```

### Создать job вручную

```bash
curl -s -X POST http://localhost:3000/api/admin/jobs \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "request_id": "'$(uuidgen)'",
    "content_type": "movie",
    "source_type": "torrent",
    "source_ref": "magnet:?xt=urn:btih:...",
    "priority": "normal"
  }' | jq .
```

### Проверка статуса job

```bash
curl -s "http://localhost:3000/api/admin/jobs/job_..." \
  -H "Authorization: Bearer $TOKEN" | jq '{status,stage,progress_percent}'
```

## 6) Deploy player на сервер (кратко)

### DNS

```text
A player.ptrack.ink -> 89.167.98.18
```

Проверка:

```bash
dig +short player.ptrack.ink
```

### Сборка и запуск контейнера

```bash
cd /opt/converter/player
cp .env.example .env
```

`/opt/converter/player/.env`:

```env
API_URL=https://api.ptrack.ink
PLAYER_KEY=<тот же ключ, что PLAYER_API_KEY в backend>
NEXT_PUBLIC_VAST_TAG=
```

Запуск:

```bash
cd /opt/converter/player
docker build -t ptrack-player:latest .
docker rm -f ptrack-player 2>/dev/null || true
docker run -d \
  --name ptrack-player \
  --restart unless-stopped \
  -p 127.0.0.1:3100:3000 \
  --env-file .env \
  ptrack-player:latest
```

Проверка:

```bash
curl -I http://127.0.0.1:3100
docker logs --tail=50 ptrack-player
```

### Nginx (идея)

- `player.ptrack.ink` -> reverse proxy на `127.0.0.1:3100`
- затем сертификат:

```bash
sudo certbot --nginx -d player.ptrack.ink
```

## 7) БД и утилиты

### Посмотреть фильмы в БД

```bash
docker compose exec -T postgres psql -U "$POSTGRES_USER" -d "$POSTGRES_DB" \
  -c "SELECT * FROM movies ORDER BY id;"
```

### Скопировать готовый фильм на сервер

```bash
scp -r /Users/robospot/prj/cleaner/converter/media/converted/mov_80baaede8740795c \
  root@89.167.98.18:/opt/converter/media/converted/
```

### Добавить фильм и субтитры в БД вручную

```bash
docker compose exec -T postgres psql -U app -d mediadb <<'EOF'
INSERT INTO movies (storage_key, tmdb_id, imdb_id, title)
VALUES ('mov_80baaede8740795c', '993988', NULL, NULL)
ON CONFLICT (storage_key) DO UPDATE
    SET tmdb_id = EXCLUDED.tmdb_id;

INSERT INTO movie_subtitles (movie_id, language, source, storage_path)
VALUES (
    (SELECT id FROM movies WHERE storage_key = 'mov_80baaede8740795c'),
    'en',
    'upload',
    'media/converted/mov_80baaede8740795c/subtitles/en.vtt'
)
ON CONFLICT (movie_id, language) DO NOTHING;
EOF
```
